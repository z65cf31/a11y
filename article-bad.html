 Web Accessibility for Content Contributors

The social model of disability is a way of thinking which states that disability is caused by barriers within society rather than an individual’s impairment. Through this lens, it’s possible to conceive that if all barriers could be removed, disability would cease to exist.

Team A11Y helps squads to remove the digital barriers and make ING websites and apps accessible to all people, with or without disabilities. The help consists of co-creation, laying out the digital baseline and provide training for accessibility champions.

Why

Our believe is that digital equality is a human right for websites, mobile applications, and digital content. But it doesn’t stop there. We need to do it for quality, innovation, compliance, relations, user experience and much much more.

03. Links and Buttons are wrongly mixed for interactivity

Choosing the wrong element will cause confusion for users, some examples:

  1. Buttons can be activated with the Spacebar, with links you will scroll down.
  2. Links appear in the link list for assistive technology, if they are buttons they will not show up.
  3. Buttons appear in the form list for assistive technology, if they are links they will not show up.
  4. After activating a link you expect the back button to work, if you activated a button this will not work.
  5. When having contact with the call center and they tell you to click a link while you only see buttons this creates confusion.

General guidelines on making the choice:

  • Use command buttons for:
    • Primary commands
    • Displaying windows used to gather input or making choices, even if they are secondary commands
    • Destructive or irreversible actions, as well as commitment within wizards and page flows
  • Use links for navigation to another page, window, or Help topic; display definitions; and command menus

07. Using styling elements or CSS to convey semantic meaning

The "digital equality is a human right" text has been made bold visually using the <b> element rather than semantically using the strong element. Also using only CSS to make text bold instead of using semantic elements would not be correct. The <figcaption> element or .figcaption class is styled italic using the CSS.

Note! Semantic mark-up such as <strong>, <em>, etc. should be used over purely visual formatting using CSS to indicate meaningful aspects such as text formatting. Additionally, structures such as lists, and headings should be used to indicate relationships of items.

As the <b> (bold) and <i> (italic) elements are both only styling elements we should avoid using those as they don't convey programmatic meaning / additional importance.

Italics are a known problem for some people with dyslexia and the general advice has been to avoid italics (particularly large blocks of italic text) and instead use bold for emphasis.

08. Links without communicating function and advanced warning when opening a new window

Links lacking a distinctive appearance using icons to communicate the function of "opening in new window".

Note! Opening pages in a new tab / window is higly discouraged whenever possible. The best practice for all links is to stay in the same tab / window as lots of people don't know what browser tabs are, have no idea pages open in new tabs and can't figure out why the back button doesn't work. This is even more often experienced on mobile.

Opening new windows automatically when a link is activated can be disorienting for people who have difficulty perceiving visual content, and for some people with cognitive disabilities, if they are not warned in advance.

When the choice is to open links in a new window all users must be informed of this function. Also users who use screen readers / assistive technology.

13. Using all capital letters for words or sentences

Sentences or phrases IN ALL CAPITALS is not a good idea. Capitalized content is more difficult to read and may give the impression that the author is shouting. Using an exclamation point or use of <em> or <strong> is often way better.

Note! Firstly, some screen readers read capitalized text letter-by-letter. And this occasionally even occurs when the HTML is in sentence-case and the CSS forces the capitalization.

The second problem is that capital letters are harder to read (for everyone, but especially people with reading disabilities).

14. Centrally aligned text is applied

Centered and left aligned text are the most widely used text alignments. How you use these text alignments can either help or harm your users when they read.

Left aligned

  • Left aligned text is easier to read
  • The text and thus the page is less noisy
  • People who zoom the page have a straight left edge
  • With horizontal scrolling the beginning of previous / next sentences are in view
  • Users can read each line by simply moving their eyes to the left edge each time
  • Paragraphs reading is faster because eyes don’t have to work as hard

Center aligned

  • When centering text the starting place of each line changes
  • Users have to work harder to find where each line begins to continue reading
  • No consistent place for moving the eyes to when completing each line
  • The text and thus the page is generally more noisy

Note! When chosing center aligned text one mistake you should never make is to combine centered text with left aligned text. This is because the unsymmetrical line lengths of the paragraph can give the headline the appearance that it’s slightly off-center. You also lose the symmetrical appearance of your paragraph. Avoid mixing the two together so that your text doesn’t look unbalanced.

The direction property specifies the text direction/writing direction which may change according to the language.

body {
  direction: ltr;
  text-align: left;
}

15. Text is fully justified

Studies have consistently shown that left-aligned text is the easiest to read contrary to justified text. How you use these text alignments can either help or harm your users when they read.

Left aligned

  • Left aligned text is easier to read
  • A ragged right margin (left-justified text) makes it easier to find the next line of text
  • Paragraphs reading is faster because eyes don’t have to work as hard

Justified text

  • Affects the readability by overflow of letters
  • Present problems for people with Dyslexia
  • Creates what’s been termed "rivers of white", referring to the spaces of white
  • Increase the need for scrolling for people who rely on zooming
  • For people with specific eye conditions or cognitive or learning difficulties the effect of kerning is that it creates uneven white spaces which make lines of print hard to follow

HTML Failure





...




body {
  direction: ltr;
  text-align: left;
}

.text-justify { text-align: justify; }

WCAG Success Criteria

  • 1.4.8 Visual Presentation Level AAA
    • F88: Failure of Success Criterion 1.4.8 due to using text that is justified (aligned to both the left and the right margins)
  • Guideline 3.1 – Readable
    • Avoiding text that is fully justified (to both left and right margins) in a way that causes poor spacing between words or characters
    • Using left-justified text for languages that are written left to right and right-justified text for languages that are written right-to-left
